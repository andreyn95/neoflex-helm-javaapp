### Задание 4.
#### Пишем свой пайплайн на GitLab

1. Зарегистрировать Gitlab Runner в кластер Kubernetes. Установить его через Helm и связать его с Вашим репозиторием на gitlab.com.
2. Написать pipeline для web-приложения из урока по Docker. В пайплайне должны быть такие стадии:
- Build. Сборка приложения в Dockerfile с помощью Kaniko, и отправка в хранилище артефактов (registry.gitlab.com или hub.docker.com)
- Deploy-on-Dev. Для деплоя используйте Helm (у вас ведь есть чарты из задания по Helm).
4. Сделайте скриншот логов успешного пайплайна.
5. Выполните команду kubectl get all в неймспейсе gitlab или в том неймспейсе, где развернулось ваше приложение. Сделайте скриншот результата.

Но сможет ли раннер использовать команды helm?

В ответ прислать.
1. Ссылку на репозиторий, из которого вы запускаете пайплайн.
2. Текст пайплайна.
3. Скриншот логов успешного пайплайна.
4. Скриншот результат выполнения команды kubectl get all в неймспейсе gitlab или в том неймспейсе, где развернулось ваше приложение.
